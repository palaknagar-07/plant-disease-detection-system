# Plant Disease Detection

TensorFlow and Streamlit project for classifying crop leaf diseases across 38 classes. The project includes a training pipeline, evaluation pipeline, single-image prediction CLI, and an interactive Streamlit demo.

## Why This Project Matters

Plant diseases can reduce crop yield when they are not detected early. This project explores image-based disease classification using convolutional neural networks and presents the model through a lightweight web app.

## Results

| Metric | Value |

| Classes | 38 |

| Test images | 10,849 |

| Final validation accuracy | 95.16% |

The full classification report and confusion matrix are generated by the evaluation pipeline. The weakest classes in the current run are `Corn_(maize)___Cercospora_leaf_spot Gray_leaf_spot` with F1 `0.777`, `Potato___healthy` with F1 `0.792`, and `Tomato___Early_blight` with F1 `0.813`.

## Features

- Custom CNN built with TensorFlow/Keras
- Reproducible train/validation split from the training folder
- Separate test-set evaluation
- Top-k prediction CLI
- Streamlit app with confidence ranking
- Saved class-name artifact to prevent label-order mismatches between training and inference
- Deployment fallback that can download the model from `PLANT_MODEL_URL`
- Clean project structure for portfolio review

## Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
make setup
```

Place the dataset in this format:

```text
dataset-2/
├── train/
│   ├── Apple___Apple_scab/
│   └── ...
└── test/
    ├── Apple___Apple_scab/
    └── ...
```

## Run The App

```bash
make app
```

The app expects `trained_model.keras` in the project root. If the model is missing, train it first, place the model in the project root, or set `PLANT_MODEL_URL` to a direct download URL for the model artifact.

## Train

```bash
make train
```

This trains the CNN using the settings in `configs/default.json`, saves the best model to `trained_model.keras`, and writes training history to `training_history.json`.
It also writes `artifacts/class_names.json`, which preserves the exact class order used during training.

## Evaluate

```bash
make evaluate
```

This evaluates the trained model on `dataset-2/test` and writes generated reports to `reports/`.

## Predict One Image

```bash
make predict IMAGE="dataset-2/test/Apple___healthy/example.jpg"
```

## Model Summary

The current model is a custom CNN with four convolution blocks, dropout regularization, and a dense classification head. It was trained for 10 epochs with Adam at a learning rate of `0.0001`.

## Limitations

- The dataset is PlantVillage-style, so many images have clean leaf framing and controlled backgrounds.
- Real-world farm images may include lighting variation, multiple leaves, soil, hands, shadows, and camera blur.
- The model predicts one of the known 38 classes and cannot reliably identify unknown diseases.
- A few classes underperform the headline accuracy, so per-class metrics should be reviewed before claiming reliability.
- Predictions should support, not replace, expert agricultural diagnosis.

## Deployment Notes

- Use Python 3.11 on Streamlit Cloud or Hugging Face Spaces; this is recorded in `runtime.txt`.
- Keep large model files out of Git. Publish `trained_model.keras` through GitHub Releases, Google Drive, or Hugging Face Hub.
- Set `PLANT_MODEL_URL` in the deployment environment when the model file is not committed.
- Use `opencv-python-headless` instead of `opencv-python` to avoid common hosted Linux display-library errors.
- Keep `artifacts/class_names.json` with the app so predictions map to the correct labels.
